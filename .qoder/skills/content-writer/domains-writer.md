# domains-writer

领域类内容撰写专家，专注撰写"全局决策"类型的技术文档。

## 适用范围

领域类主题（domains/*），如前端开发、后端开发、技术管理等。

## 写作重点

| 维度 | 要求 |
|------|------|
| 选型依据充分 | 有对比表格或决策矩阵，判断标准清晰 |
| 技术关系清晰 | 技术栈关系有说明，概念依赖明确 |
| 决策过程完整 | 案例分析有完整的决策过程和权衡 |
| 前沿性 | 关注当前趋势，避免过时信息 |

---

## 详解写作结构

### 是什么
- 一句话定义 + 类比
- 在整个技术领域中的位置

### 对比/关键环节
- 该概念的核心组成部分
- 与相关概念的区别和联系
- 用表格对比不同方案

### 选型依据/决策影响
- 理解这个概念对技术决策有什么帮助
- 什么情况下选择什么方案
- 决策矩阵

---

## 示例

```markdown
## 前端框架选型

### 是什么
前端框架选型是在 React、Vue、Angular 等框架中做出技术决策的过程，就像"选车"——根据需求选轿车、SUV 还是跑車。

### 主流框架对比

| 维度 | React | Vue | Angular |
|------|-------|-----|---------|
| 学习曲线 | 中等 | 平缓 | 较陡 |
| 生态规模 | 最大 | 大 | 大 |
| 适用场景 | 大型应用 | 中小型、快速开发 | 企业级 |
| TypeScript | 支持 | 支持 | 默认 |

### 选型依据

**选 React 当**：
- 团队有 React 经验
- 需要最大生态支持
- 项目规模大、长期维护
- 需要跨平台（React Native）

**选 Vue 当**：
- 团队经验较少
- 追求快速开发
- 项目中等规模
- 需要渐进式采用

**选 Angular 当**：
- 企业级应用
- 需要完整方案（Router/HTTP/表单内置）
- 团队有 Java 背景（TypeScript 友好）
- 大型团队需要统一规范
```

### 决策结果
- 最终选择

## 示例

```markdown
## 场景一：中型电商项目技术选型

### 需求背景
- 电商平台，预计日活 10 万
- 团队 5 人，前端 2 人
- 3 个月上线 MVP，后续持续迭代

### 候选方案对比

| 方案 | 优势 | 劣势 |
|------|------|------|
| React + Next.js | 生态成熟、SSR 支持 | 学习成本较高 |
| Vue + Nuxt | 上手快、开发效率高 | 生态稍弱 |
| Angular | 完整方案、适合大团队 | 学习曲线陡、过于重 |

### 决策结果
选择 **Vue + Nuxt**：
- 团队前端人手少，需要快速上手
- Vue 生态足够支撑电商场景
- Nuxt 提供开箱即用的 SSR

### 经验总结
- 小团队优先考虑开发效率
- 不要过度追求"最佳实践"，够用就好
- 预留技术债务清理的时间
```

## 速查表格式

选型决策速查表：

```markdown
| 场景 | 推荐方案 | 关键理由 |
|------|----------|----------|
| 大型团队、长期项目 | React/Angular | 生态成熟、规范完善 |
| 小团队、快速迭代 | Vue | 上手快、开发效率高 |
| 企业级、需要完整方案 | Angular | 内置完整、统一规范 |
```

## 约束

- 对比表格必须有明确的判断维度
- 决策理由要具体，不能"都很好"
- 案例必须有完整的决策过程
- 关注当前趋势，避免过时建议
