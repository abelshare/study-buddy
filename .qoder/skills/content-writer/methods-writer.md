# methods-writer

方法论类内容撰写专家，专注撰写"迁移应用"类型的知识文档。

## 适用范围

方法论类主题（methods/*），如费曼学习法、根因分析、第一性原理等。

## 写作重点

| 维度 | 要求 |
|------|------|
| 步骤清晰 | 每步是一个可执行动作，不能模糊 |
| 场景明确 | 必须说明适用和不适用场景 |
| 误区覆盖 | 至少 2-3 个常见误区 |
| 可迁移 | 读者能迁移到自己的场景 |

---

## 详解写作结构

### 是什么
- 这一步的定义和目的
- 在整个方法中的位置

### 怎么做
- 具体操作方法
- 关键要点（2-3 个）
- 可选：示例说明

### 常见误区
- 执行时容易犯的错误
- 如何避免

---

## 示例

```markdown
## 步骤二：教授新手

### 是什么
用简单语言向"假想的新手"解释概念，检验自己是否真正理解。这是费曼学习法的核心环节。

### 怎么做

1. **假设听众零基础**
   - 不使用专业术语
   - 用生活类比解释

2. **边讲边写**
   - 写下来比口头说更能暴露漏洞
   - 可以用白板、博客、笔记

3. **控制时间**
   - 每个概念不超过 5 分钟
   - 太长说明概念太大，应拆分

### 常见误区

| 误区 | 后果 | 正确做法 |
|------|------|----------|
| 使用术语 | 掩盖理解漏洞 | 全程用大白话 |
| 选太泛的概念 | 讲不清楚 | 拆成小概念逐个攻克 |
| 只看不练 | 以为自己懂了 | 必须开口/动笔讲出来 |
| 完美主义 | 卡在第一步 | 先讲再迭代改进 |
```

## 场景应用写作结构

### 场景描述
- 具体场景是什么
- 为什么适合用这个方法

### 应用步骤
- 如何在这个场景中应用方法
- 每步的具体操作

### 关键要点
- 这个场景需要注意什么
- 如何迁移到类似场景

---

## 示例

```markdown
## 场景一：学习新编程语言

### 场景描述
你想学习 Rust，但看了一堆教程还是云里雾里。这时候用费曼学习法可以快速检验理解程度。

### 应用步骤

1. **选择概念**：不要选"Rust"，太泛了。选"Rust 的所有权机制"或"Rust 的生命周期"。

2. **教授新手**：假设听众只会 Python，试着解释"为什么 Rust 不需要垃圾回收"。
   - 用类比：就像租房，租客搬走时必须交还钥匙，不能一直占着

3. **识别盲区**：发现"借用检查器"讲不清楚 → 说明这块没真懂

4. **简化类比**：把借用检查器比作"门卫"，检查每个访问是否合规

### 关键要点
- 选择概念时，从小处入手
- 类比要贴近目标听众的背景
- 讲不清楚的地方就是你要补的课
```

## 速查表格式

步骤清单：

```markdown
## 费曼学习法步骤清单

- [ ] 选择一个具体概念
- [ ] 用大白话向假想新手解释
- [ ] 记录卡壳的地方
- [ ] 回去重学卡壳点
- [ ] 用类比简化解释

**适用场景**：新概念学习、知识检验、技术分享准备
**不适用场景**：纯记忆性内容、技能型操作
```

---

## 约束

- 步骤必须是可执行的动作
- 不能用模糊描述（如"理解概念"、"多练习"）
- 必须说明适用和不适用场景
- 常见误区要典型、可避免
- 场景应用要展示迁移思路
