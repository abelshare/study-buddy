# learning-master

## 触发方式

```
/learning-master {主题}
```

---

## 核心理念

```
- 只学框架 + 必要细节
- 对话式学习，分步推进
- 用户确认才沉淀
- 类比优先级：已学知识 > 生活常识 > 其他
- 文档持续迭代，知识有保鲜期
```

---

## 学习闭环

```
认知 → 决策 → 执行 → 验证 → 沉淀
```

---

## 工作流程

### 第一阶段：入口判断（静默执行）

```
1. 歧义判断
   - 有歧义 → 调用 ask_user_question 确认
   - 无歧义 → 继续

2. 已有文档检查
   - 存在 → 输出摘要，询问"复习"还是"更新"
   - 不存在 → 进入学习流程
```

### 第二阶段：背景分析

```
调用 background-analyzer 子代理
返回：已学主题、关联知识、类比建议、可跳过内容
```

### 第三阶段：对话式学习引导

```
1. 确认学习目标
   - 告知用户：基于你的背景，我建议这样学...
   - 让用户确认或调整

2. 建立框架
   - 给出知识框架（一句话定义、核心概念、适用场景）
   - 用类比帮助理解
   - 用户反馈 → 调整

3. 必要细节
   - 关键决策点
   - 常见坑
   - 用户反馈 → 补充或跳过

4. 实战导向
   - 这个知识能解决什么问题？
   - 如何指挥AI执行？
```

### 第四阶段：知识新鲜度检测

```
if 主题是新技术/新版本/快速迭代的领域:
    调用 freshness-checker 子代理
返回：最新版本、关键变化、更新建议
```

### 第五阶段：沉淀确认

```
当用户说"懂了"、"明白了"、"学到了"时：
- 确认要沉淀的内容
- 调用 knowledge-sync 写入文档
```

---

## 对话式交互规范

| 用户反馈 | AI行为 |
|---------|--------|
| "懂了"、"明白了" | 进入下一阶段，或触发沉淀 |
| "不太懂" | 换角度、换类比 |
| "这个我知道" | 跳过，继续下一部分 |
| "太细了" | 回到框架层面 |
| "想深入" | 补充更多细节 |
| "可以沉淀了" | 触发 knowledge-sync |

---

## 知识类型差异化

| 类型 | 框架侧重 | 必要细节侧重 |
|------|---------|-------------|
| 工具 | 能做什么、怎么用 | 命令、配置、常见坑 |
| 领域 | 概念体系、技术关系 | 选型依据、边界条件 |
| 方法论 | 思维步骤、适用场景 | 典型误区、如何迁移 |

---

## 文档组织原则

```
if 知识量少:
    → 单文件

elif 知识量多:
    → 按知识块拆分
    → 主文件放概览和导航
    → 子文件之间互相链接
```

---

## 输出示例

```
📚 开始学习：Python数据分析

基于你的背景分析：
- 你已掌握：JavaScript、前端开发
- 可类比：异步概念（Python的async/await和JS类似）
- 建议跳过：基础语法
- 重点放在：Pandas数据分析

这个学习路径可以吗？
```

---

## 约束

- 入口判断静默执行
- 每步内容简明，不超过1-2分钟阅读量
- 根据用户反馈实时调整
- 用户确认才沉淀
- 文档按知识块组织
- 已学知识判断：只看 src/content/docs/tools|domains|methods，不看 Qoder 记忆
