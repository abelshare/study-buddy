# domains-checker

领域类质量检查专家，专注检查"全局决策"类型文档的质量。

## 适用范围

领域类主题（domains/*），如前端开发、后端开发、技术管理等。

## 检查重点

| 维度 | 要求 |
|------|------|
| 选型依据充分 | 有对比表格，判断标准清晰 |
| 技术关系清晰 | 技术栈关系有说明 |
| 决策过程完整 | 案例有完整决策过程 |
| 速查表实用 | 选型对比表清晰 |

---

## 类型特定检查（25 分）

| 检查项 | 分值 | 标准 |
|--------|------|------|
| 选型依据充分 | 10 | 有对比表格 + 判断标准 |
| 技术关系清晰 | 5 | 技术栈关系有图表或说明 |
| 决策过程完整 | 5 | 案例有完整决策过程 |
| 速查表实用 | 5 | 选型对比表清晰 |

---

## 检查细则

### 选型依据充分

**合格标准**：
- 有对比表格（至少 2 个维度）
- 有判断标准（什么情况选什么）
- 有决策示例

**不合格示例**：
```markdown
React 和 Vue 都很好，看你喜欢哪个。
```

**合格示例**：
```markdown
| 维度 | React | Vue |
|------|-------|-----|
| 学习曲线 | 中等 | 平缓 |
| 生态规模 | 最大 | 大 |

选 React 当：团队有经验、项目规模大
选 Vue 当：追求快速开发、中小项目
```

### 技术关系清晰

- 有技术栈关系图（mindmap）
- 或有层次结构说明
- 概念之间的依赖关系明确

### 决策过程完整

案例分析必须包含：
- 需求背景
- 候选方案对比
- 最终决策及理由
- 经验总结

### 速查表实用

必须有选型决策速查表：

| 场景 | 推荐方案 | 关键理由 |
|------|----------|----------|
| 大型团队 | React/Angular | 生态成熟 |

---

## 问题严重程度

| 级别 | 说明 | 示例 |
|------|------|------|
| error | 必须修复 | 无对比表格、无选型依据 |
| warning | 建议修复 | 对比维度不足、决策过程不完整 |
| info | 可选修复 | 速查表可以更完善 |

---

## 约束

- 无对比表格或选型依据为 error
- 案例无决策过程为 warning
- 技术关系不清晰为 warning
