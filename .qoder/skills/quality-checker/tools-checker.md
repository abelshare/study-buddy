# tools-checker

工具类质量检查专家，专注检查"上手操作"类型文档的质量。

## 适用范围

工具类主题（tools/*），如 Docker、Git、VS Code 等。

## 检查重点

| 维度 | 要求 |
|------|------|
| 命令可运行 | 示例命令语法正确 |
| 常见坑覆盖 | 至少 2-3 个典型陷阱 |
| 注释去 AI 味 | 注释描述意图，非语法旁白 |
| 速查表实用 | 覆盖高频命令 |

---

## 类型特定检查（25 分）

| 检查项 | 分值 | 标准 |
|--------|------|------|
| 命令可运行 | 10 | 示例命令语法正确，参数合理 |
| 常见坑覆盖 | 5 | 至少 2-3 个典型陷阱 |
| 注释去 AI 味 | 5 | 注释描述意图/场景 |
| 速查表实用 | 5 | 覆盖高频命令 |

---

## 检查细则

### 命令可运行

**合格标准**：
- 命令语法正确
- 参数使用合理
- 可直接复制执行

**不合格示例**：
```bash
# 错误：参数不存在
docker run --non-existent-flag nginx
```

**合格示例**：
```bash
# 正确
docker run -d -p 8080:80 nginx
```

### 常见坑覆盖

必须包含至少以下 2 类：
- 安装/配置坑
- 使用过程坑
- 边界条件坑
- 版本兼容坑

### 注释去 AI 味

**❌ 不合格**：
```bash
# 定义变量
IMAGE_NAME="my-app"
# 循环
for f in *.js; do echo $f; done
```

**✅ 合格**：
```bash
# 镜像名称，用于构建和推送
IMAGE_NAME="my-app"
# 检查所有 JS 文件语法
for f in *.js; do echo $f; done
```

### 速查表实用

必须覆盖核心命令的高频用法：

| 命令 | 用途 | 常用参数 |
|------|------|----------|
| docker run | 运行容器 | -d -p -v |
| docker build | 构建镜像 | -t -f |

## 问题严重程度

| 级别 | 说明 | 示例 |
|------|------|------|
| error | 必须修复 | 命令语法错误 |
| warning | 建议修复 | 常见坑覆盖不足 |
| info | 可选修复 | 速查表可以更完善 |

---

## 约束

- 命令必须可运行，语法错误为 error
- 常见坑少于 2 个为 warning
- 注释有语法旁白为 warning
