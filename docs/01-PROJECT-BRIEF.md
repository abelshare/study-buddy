# StudyBuddy 项目简介

> 版本：v1.0  
> 更新日期：2026-02-24  
> 项目代号：StudyBuddy

---

## 1. 项目愿景

**一句话定义**：AI 驱动的个人知识成长伙伴，将碎片化学习转化为结构化知识体系。

> **核心理念：AI 时代，会管比会做更有直接价值**

---

## 2. 核心价值主张

### 2.1 解决的问题

| 痛点 | 传统方式 | StudyBuddy 方案 |
|------|----------|-----------------|
| 知识碎片化 | 笔记散落各处 | 统一知识库 + 自动分类 |
| 学习无体系 | 想到哪学到哪 | 三阶段学习框架（概览→详解→实战） |
| 内容生成慢 | 手动整理耗时 | AI 自动生成结构化文档 |
| 复习效率低 | 全文重读 | 速查表 + 思维导图快速回顾 |

### 2.2 核心理念

- **从记忆转向检索**：不再死记硬背，而是快速理解框架，需要时精准检索
- **从深度转向广度**：了解整体即可，知道"何时用"比"怎么用"更重要
- **从线性转向网状**：知识点之间的关联比单个知识点本身更有价值
- **从执行转向管理**：以管理者视角学习，关注决策而非细节实现

---

## 3. 目标用户

### 3.1 用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 技术管理者 / 终身学习者 |
| 视角 | **管理者思维**，关注"为什么"和"何时用"，不深入实现细节 |
| 习惯 | 使用 AI 编程工具辅助学习（Qoder） |
| 偏好 | 简洁美观、快速理解、拒绝信息过载 |
| 目标 | 建立知识体系，需要时能快速检索和应用 |

### 3.2 核心诉求

| 优先级 | 诉求 | 说明 |
|--------|------|------|
| P0 | 快速建立知识体系 | 通过脑图/思维链快速了解全局 |
| P0 | 实用导向 | 关注应用场景，不深入实现细节 |
| P1 | 视觉记忆点 | 美观的页面设计，助力记忆 |
| P1 | 速查能力 | 需要时能快速找到关键信息 |
| P2 | 知识关联 | 了解知识点之间的联系 |

## 4. 技术栈一览

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| **框架** | Astro | 静态优先、性能极致、部署简单 |
| **主题** | Starlight | 开箱即用文档站、内置搜索/导航/代码高亮 |
| **图表** | Mermaid | Markdown 原生语法、AI 直接生成 |
| **样式** | Starlight 主题 + 自定义 CSS | 降低复杂度 |
| **AI 工具链** | Qoder（Skills + MCP） | 多代理协作、可扩展能力 |
| **内容格式** | 纯 Markdown | 版本控制友好、可移植 |

---

## 5. 项目范围

### 5.1 做什么（In Scope）

- 基于学习主题自动生成三阶段结构化文档
- 静态文档站点，支持搜索和导航（本地使用）
- Mermaid 图表可视化（思维导图、流程图）
- 多 Agent 协作的 AI 生成工作流
- 文档分类体系（工具/领域/方法论）

### 5.2 不做什么（Out of Scope）

| 功能 | 原因 |
|------|------|
| 用户登录系统 | 个人工具，无需多用户 |
| 数据库存储 | 纯文件系统，Git 版本控制 |
| 实时协作编辑 | 避免复杂度 |
| 内容推荐算法 | 保持简单 |
| 移动端 App | 优先 Web 端 |
| 线上部署 | 本地使用为主 |
| 学习进度追踪 | 非核心功能 |

## 6. 与竞品差异

| 特性 | Obsidian | Notion | StudyBuddy |
|------|----------|--------|------------|
| AI 生成 | 插件支持 | 内置 AI | **全自动生成** |
| 学习框架 | 无 | 无 | **三阶段学习法** |
| 使用方式 | 本地 | SaaS | **本地静态站点** |
| 维护成本 | 低 | 中 | **零维护** |
| 数据所有权 | 本地 | 云端 | **本地 + Git** |
| 可视化 | 图谱 | 看板 | **Mermaid 图表** |
| 学习视角 | 通用 | 通用 | **管理者视角** |
