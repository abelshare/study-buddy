# StudyBuddy 功能需求规格

> 版本：v1.0  
> 更新日期：2026-02-24

---

## 1. 需求优先级说明

采用 **MoSCoW** 方法标记优先级：

| 标记 | 含义 | MVP 阶段 |
|------|------|----------|
| **M** (Must) | 必须实现，核心功能 | 包含 |
| **S** (Should) | 应该实现，重要功能 | 部分包含 |
| **C** (Could) | 可以实现，增强功能 | 不包含 |
| **W** (Won't) | 本期不做 | 不包含 |

---

## 2. 功能需求清单

### 2.1 文档站基础 [M]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-001 | Astro + Starlight 项目初始化 | 本地 `npm run dev` 可正常启动 |
| REQ-002 | 自定义主题配色 | 配色方案与品牌一致，支持深色模式 |
| REQ-003 | 首页设计 | 包含项目介绍、快速导航、最近更新 |
| REQ-004 | 文档分类目录结构 | 支持 3 大分类：tools/（工具）、domains/（领域）、methods/（方法论） |

### 2.2 内容展示 [M]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-005 | Markdown 渲染 | 支持 GFM 语法，表格、代码块正常显示 |
| REQ-006 | 代码高亮 | 支持 20+ 编程语言语法高亮 |
| REQ-007 | 内置搜索 | Starlight 默认搜索可用，支持中文 |
| REQ-008 | 响应式布局 | 桌面端、平板、移动端自适应 |

### 2.3 可视化 [M]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-009 | Mermaid 图表集成 | 支持 flowchart、mindmap、sequenceDiagram |
| REQ-010 | 思维导图渲染 | mindmap 语法可正确渲染为可视化图表 |
| REQ-011 | 速查表样式 | 提供专用 CSS 类，表格样式美观紧凑 |

### 2.4 AI 生成管道 [M]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-012 | learning-master Skill | 主控 Skill 可接收主题输入并协调子 Skill |
| REQ-013 | topic-analyzer Skill | 输出标准化 JSON 分析结果 |
| REQ-014 | outline-planner Skill | 生成符合三阶段框架的大纲 |
| REQ-015 | content-writer Skill | 分段生成概览/详解/实战内容 |
| REQ-016 | visual-designer Skill | 自动生成至少 2 个 Mermaid 图表 |
| REQ-017 | quality-checker Skill | 检查通过率 >= 80% 才输出 |
| REQ-018 | 一键生成命令 | `/learn {topic}` 命令可触发完整流程 |
| REQ-019 | **MCP 集成：Context7** | 调用 Context7 查询官方文档，确保 API 和版本信息准确 |
| REQ-020 | **MCP 集成：WebSearch** | 联网搜索最新资讯和最佳实践 |
| REQ-021 | **MCP 集成：WebFetch** | 抓取官方网站内容作为参考来源 |
| REQ-022 | **数据来源标注** | 生成内容必须标注数据来源（如：React 官方文档 v18.2） |

### 2.5 学习框架支持 [S]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-023 | 概览阶段内容 | 包含：一句话定义、核心问题、适用场景、前置知识、思维导图 |
| REQ-024 | 详解阶段内容 | 每个概念包含：是什么、为什么、怎么用 |
| REQ-025 | 实战阶段内容 | 按难度分级：初级/中级/高级 |
| REQ-026 | 速查表组件 | 每篇文档包含可快速查阅的 Cheatsheet |

### 2.6 增强功能 [C]

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| REQ-027 | 知识图谱可视化 | 展示文档间的概念关联 |
| REQ-028 | 批量生成多篇文档 | 支持传入主题列表批量生成 |
| REQ-029 | 知识关联自动发现 | AI 自动识别并链接相关文档 |

### 2.7 本期不做 [W]

| ID | 需求描述 | 原因 |
|----|----------|------|
| REQ-W01 | 用户登录系统 | 个人工具无需多用户 |
| REQ-W02 | 评论/讨论功能 | 增加后端复杂度 |
| REQ-W03 | Anki 卡片导出 | 非核心功能 |
| REQ-W04 | 多语言国际化 | 优先中文内容 |

## 3. 非功能性需求

### 3.1 性能需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-001 | 文档生成时间 | < 30 秒/篇 |
| NFR-002 | 站点构建时间 | < 60 秒（100 篇文档） |
| NFR-003 | 页面加载时间 | < 2 秒（首屏） |
| NFR-004 | Lighthouse 性能分 | >= 90 |

### 3.2 可维护性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-005 | 内容格式 | 纯 Markdown，无私有格式 |
| NFR-006 | 版本控制 | 所有内容 Git 管理 |
| NFR-007 | 依赖管理 | package.json 锁定版本 |

### 3.3 可移植性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-008 | 文档独立性 | 生成的 Markdown 可在任何阅读器打开 |

### 3.4 安全性需求

| ID | 需求描述 | 指标 |
|----|----------|------|
| NFR-010 | 无敏感数据 | 不存储用户隐私信息 |
| NFR-011 | 依赖安全 | 无已知高危漏洞依赖 |

## 5. 约束与假设

### 5.1 约束条件

- 使用 Qoder 作为 AI 编程环境
- 不引入后端服务器，纯静态站点
- 开发环境为 Windows

### 5.2 假设条件

- Qoder Skill 支持多代理协作
- Mermaid 图表渲染无兼容性问题
- 用户具备基本的 Markdown 编辑能力
